# UX Specification: Screen 3 — Level 2 (FCP Detail)

> Parent screen: [03-Maps view](./03-Maps%20view.md) · Next level: [03.2 — TrenchImage Detail](./03.2-trenchimage-view.md)

## 3. Level 2 — FCP Detail

### 3.1 Map Behavior

When an FCP is selected, the map zooms to fit the selected **FCP polygon** from `*_FCP_Polygons.geojson` (e.g. `F012 [81]` in `kmlDescriptionSimple`). The polygon is outlined prominently; other FCP polygons are de-emphasized.

Inside the selected FCP area:

- **Trench lines** from `*_Trenches.geojson` that intersect or fall within the polygon are rendered with status colours (Green / Yellow / Red), aggregated from linked **`PhotoAnalysis`** rows on images assigned to this FCP.
- **Photo markers** appear for each uploaded image assigned to this FCP (upload-time FCP declaration — see [`docs/problem_reasoning.md`](../problem_reasoning.md)). Markers use the photo’s documentation status colour.

Trench segment lines may show a **tooltip on hover** (segment name, length, status) but do **not** trigger a deeper map level on click.

**Optional overlay:** Expected photo slots (~every 5 m along trench geometry) may be drawn as small dots for gap analysis. These are **not** the primary navigation unit; the user navigates by **photo markers**, not abstract segment nodes.

**Data model:** Per-FCP aggregates are **rollups** over `PhotoAnalysis` rows for images in that FCP plus trench geometry clipped to the polygon — not a separate FCP analysis table in phase 1.

### 3.2 Side Panel — FCP Summary

A panel slides in smoothly from the right edge of the screen. The map remains fully visible and interactive to the left of the panel. The panel contains:

- **FCP Name / ID:** From GeoJSON `fcpName` / `kmlName` (e.g. `F012`, `F175 schacht`)
- **Project / Cluster:** `clusterName` (e.g. `CLP20417A`) — same entity as the dashboard project
- **Homes / Buildings:** From `countHomes`, `countBuildings` where present (e.g. `81 homes • 41 buildings`)
- **Trench length in FCP:** Sum of trench segment lengths inside the polygon (e.g. `2.1 km`)
- **Photo count:** e.g. `47 photos documented, 52 expected`
- **Status breakdown for this FCP:** Mini Green / Yellow / Red bar for this FCP only
- **Instruction text:** "Click a photo on the map or use the arrows below to review individual trench images."
- **Left / Right Navigation Arrows:** Step through photos in this FCP sequentially. Each step updates the map marker highlight and can enter Level 3 when the user opens full TrenchImage detail.

### 3.3 Exiting Level 2

The user returns to Level 1 (Project overview) by either:

- Clicking the "← Back" button at the top of the side panel.
- Clicking anywhere on the map outside the currently selected FCP polygon.

Entering Level 3: See [03.2 — TrenchImage Detail](./03.2-trenchimage-view.md).
