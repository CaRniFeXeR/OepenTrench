# UX Specification: Screen 2 — Project Images Panel

> Parent screen: [02 — File Upload](./02-File%20Upload.md) · Analysis tag definitions: [03.2 — TrenchImage Detail](./03.2-trenchimage-view.md)

## Context for UI Engineer

The project detail page (Screen 2) shows upload controls and a live map preview in a two-column layout. Below that layout, a full-width **Project Images** panel lists every uploaded trench photo with AI analysis results. It gives reviewers a scannable inventory without opening the map drill-down (Screen 3).

---

## 1. Placement

- Renders at the **bottom** of the project detail page, below the upload + map grid.
- Visible as soon as the project loads (including while uploads are in progress).
- Uses the same page width as the main content (`max-w-6xl`).

See also: [02 — File Upload](./02-File%20Upload.md) layout.

---

## 2. Purpose

- Browse all project photos in one table.
- See documentation quality (Good / Poor / Missing / Pending) and per-tag analysis at a glance.
- Filter and paginate large projects (up to 1,000+ photos) without leaving Screen 2.

---

## 3. Toolbar

Above the table, left to right (wraps on narrow screens):

| Control | Options | Behaviour |
| --- | --- | --- |
| **Search** | Free text | Case-insensitive filter on filename (`original_label`). |
| **Quality** | All, Good, Poor, Missing, Pending analysis | Client-side filter on effective documentation category. |
| **Sort** | Newest first (default), Filename A–Z, Filename Z–A | `created_at` or label sort. |
| **Per page** | 25 (default), 50, 100 | Page size for client-side pagination. |

Changing any filter, sort, or page size resets to **page 1**.

### Quality filter rules

| Filter | Match |
| --- | --- |
| Good | Effective category `green` (`reviewer_override_category` ?? `category`) |
| Poor | Effective category `yellow` |
| Missing | Effective category `red` or unknown (analysis exists but no satisfactory category) |
| Pending analysis | No `PhotoAnalysisRead` row yet (`analysis` is null) |

---

## 4. Table

White card with header (“Project images”) and photo count subtitle.

| Column | Content |
| --- | --- |
| **Preview** | 48×48 thumbnail from `GET /projects/{project_id}/images/{asset_id}/content` |
| **Filename** | `original_label`, truncated with tooltip on hover |
| **Uploaded** | `created_at` as locale date |
| **Quality** | Badge — see below |
| **Analysis** | Compact tag row (short labels) or “Analysis pending” |

### Quality badges

| Badge | Colour | When |
| --- | --- | --- |
| Good | Emerald | Effective category `green` |
| Poor | Amber | Effective category `yellow` |
| Missing | Red | Effective category `red` or unknown |
| Pending | Slate | No analysis yet |

### Analysis tags

Same seven checks as [03.2 — TrenchImage Detail](./03.2-trenchimage-view.md), shown with short labels in the table (Duct, Sand, Ruler, Seal, GPS, Date, Privacy). Green check / red cross per tag.

Field mapping and API names: see the **Technical Description** table in 03.2.

---

## 5. Pagination

- **Client-side** over all image assets returned in `GET /projects/{project_id}` → `assets[]` where `kind === "image"`.
- Footer: “Showing 26–50 of 312” (and “filtered from N” when filters reduce the set).
- **Previous** / **Next** buttons; disabled on first/last page.
- “Page X of Y” between buttons.

---

## 6. Data source

| Data | Source |
| --- | --- |
| Asset list + analysis | `GET /projects/{project_id}` → `ProjectDetailRead.assets` |
| Thumbnail / image bytes | `GET /projects/{project_id}/images/{asset_id}/content` (relative URL, proxied in dev) |
| Analysis fields | Nested `PhotoAnalysisRead` on each image asset |

No dedicated list-images endpoint in v1. The panel refreshes when the parent page reloads project detail (e.g. after upload completes).

---

## 7. Empty and edge states

| State | Copy |
| --- | --- |
| No photos uploaded | “Upload photos above to see them listed here with analysis results.” |
| Filters match nothing | “No photos match your filters.” |
| Row without analysis | Quality badge **Pending**; analysis column “Analysis pending” |

---

## 8. Out of scope (v1)

- Row click to open map / TrenchImage detail
- False-call override from the table
- Server-side `GET /projects/{project_id}/images` with `limit` / `offset`
- Bulk actions (export, re-analyse)

---

## 9. Developer notes

- Reuse shared analysis UI (`qualityBadge`, `AnalysisTagRow`) with map trench detail panel for consistent badges and tags.
- List panel passes `pendingWhenNull: true` on quality badges so “Pending” differs from “Missing” (map panel keeps “Missing” when analysis is absent).
