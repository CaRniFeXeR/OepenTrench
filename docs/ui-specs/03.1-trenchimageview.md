# UX Specification: Screen 3 — Level 2 (Trench Detail)

> Parent screen: [03-Maps view](./03-Maps%20view.md) · Next level: [03.2 — Photo Navigation](./03.2-individual_images.md)

## 3. Level 2 — Trench Detail

### 3.1 Map Behavior

When a trench is selected, the map zooms in to show the individual trench line in detail. The route line is now rendered with individual nodes visible along it — one node approximately every 5 metres, representing each expected photo position. Nodes are color-coded to match their photo status (Green / Yellow / Red).

### 3.2 Side Panel — Trench Summary

A panel slides in smoothly from the right edge of the screen. The map remains fully visible and interactive to the left of the panel. The panel contains:

- **Trench Name / ID:** (e.g., "Segment 04 — FCP2 to FCP3")
- **Total Length:** (e.g., "320m")
- **Node Count:** (e.g., "64 nodes expected, 58 documented")
- **Status Breakdown for this trench:** Mini version of the Green/Yellow/Red bar.
- **Instruction text:** "Click a node on the map or use the arrows below to review individual photos."
- **Left / Right Navigation Arrows:** Allow the user to step through photos sequentially without clicking the map. Clicking an arrow updates the map highlight and the panel content simultaneously.

### 3.3 Exiting Level 2

The user returns to Level 1 (Cluster Overview) by either:

- Clicking the "← Back" button at the top of the side panel.
- Clicking anywhere on the map outside the currently selected trench.

Entering Level 3: See [03.2 — Photo Navigation](./03.2-individual_images.md).
