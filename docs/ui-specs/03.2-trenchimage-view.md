# UX Specification: Screen 3 — Level 3 (TrenchImage Detail)

> Parent screen: [03-Maps view](./03-Maps%20view.md) · Previous level: [03.1 — FCP Detail](./03.1-fcp-view.md)

## 4. Level 3 — TrenchImage Detail

### 4.1 Trigger

The user enters Level 3 by either:

- Clicking a **photo marker** within the selected FCP on the map.
- Pressing the left or right arrow in the Level 2 FCP side panel to step to the previous or next photo in that FCP.

### 4.2 Map Behavior

The map zooms in further to center on the active photo marker. The selected marker is visually highlighted (e.g., enlarged with a white ring or pulsing animation). Other photo markers in the same FCP remain visible but de-emphasized. The FCP polygon outline and relevant trench lines stay visible for context.

### 4.3 Side Panel — TrenchImage Detail

The side panel updates to show the individual trench image and its analysis. The panel layout from top to bottom:

#### Photo Viewer

A large photo display area showing the trench image for the active marker. Left and right arrow buttons sit on either side of the photo for navigation within the current FCP.

#### Photo Quality Label

A simple, prominent status tag directly below the photo:

- ✅ **Good**
- ⚠ **Poor**
- ✗ **Missing**

#### Technical Description (Fixed Tags)

A set of fixed-label tags showing what the AI detected (or failed to detect) in the photo. Each tag is either confirmed (green checkmark) or flagged (red cross):

| Tag | Description |
| --- | --- |
| Duct Visible | Conduit pipe is clearly visible in the trench |
| Sand Bedding | Sand cushion layer is present beneath the cable |
| Burial Depth / Ruler | Measurement ruler is readable and depth is compliant |
| Pipe End Seal | Duct end cap is visible and in place |
| GPS Match | Photo GPS coordinates match the expected route segment |
| Date Valid | Photo timestamp falls within the construction window |
| Privacy Clear | No faces or license plates visible in the photo |

**API / persistence (canonical):** Per image, the backend stores a **`PhotoAnalysis`** row (OpenAPI: `PhotoAnalysisRead`) keyed by **`asset_id`**. Use the following field names in clients (JSON matches Python `snake_case`).

**Phase 1 — persisted now**

| Field | Role |
| --- | --- |
| `is_in_domain` | Photo belongs to the trench documentation domain |
| `has_white_paper` | Related to sand / bedding visibility (see UX “Sand Bedding”; refine copy if product differentiates) |
| `has_ruler` | Burial depth / ruler legible (with optional `estimated_depth`) |
| `estimated_depth` | Optional numeric depth estimate |
| `has_duct` | Duct visible |
| `estimate_number_of_ducts` | Optional duct count |
| `has_gdpr_problems` | **Inverse of “Privacy Clear”** — `false` means privacy OK, `true` means flagged |
| `is_duplicated` | Duplicate detection |
| `category` | Optional `green` \| `yellow` \| `red` — documentation quality for map rollups |

**UI ↔ `category` / badge:** **Good** ↔ `green`; **Poor** ↔ `yellow`; **Missing** ↔ no satisfactory documentation (**`category` null**, or treat **`red`** explicitly — pick one convention in the client and keep it consistent).

**Phase 2 — also persisted (extended tags & review)**

| UX tag | Field |
| --- | --- |
| Sand Bedding | `has_sand_bedding` |
| Pipe End Seal | `has_pipe_end_seal` |
| GPS Match | `gps_matches_route` |
| Date Valid | `date_valid` |
| False Call / human override | `is_false_call`, optional `reviewer_override_category` |

#### False Call Button

A clearly labelled button at the bottom of the panel: **"Flag as False Call"**. This allows the human reviewer to override the AI classification for this specific photo. Clicking it opens a small inline confirmation: "Mark this photo as manually reviewed and override AI result?" with Confirm and Cancel options.

#### Navigation Footer

- "← Previous Photo" and "Next Photo →" text links or icon buttons at the bottom of the panel.
- A counter scoped to the current FCP (e.g., "Photo 12 of 47 in F012").

### 4.4 Exiting Level 3

The user returns to Level 2 (FCP Detail) by either:

- Clicking the "← Back to FCP" button at the top of the side panel.
- Clicking anywhere on the map outside the active photo marker but still within the same FCP area.

## 5. Developer Notes for Cursor / AI Assistant

- **Mapping Library:** Use Mapbox GL JS, Leaflet, or Google Maps API. The library must support GeoJSON rendering, custom line and polygon styling, clickable FCP polygons, photo markers, and programmatic zoom/pan transitions between levels.
- **Smooth Zoom Transitions:** Level transitions (1 → 2 → 3) should use smooth animated zoom and pan, not hard jumps. This is the core of the "Google Maps feel" interaction model.
- **Side Panel:** Build as a reusable sliding panel component with three slot states: `hidden`, `fcpSummary`, `trenchImageDetail`. Use CSS `transform: translateX()` for the slide-in/out animation.
- **Photo markers:** Hidden at Level 1 (project overview) to avoid clutter. Shown at Level 2 and Level 3 when zoomed into an FCP. Optional expected-slot dots (~5 m) may share the same visibility rules if enabled.
- **Left/Right Arrow Navigation:** Arrows in the FCP panel and TrenchImage viewer must stay synchronized with the map — pressing an arrow updates the highlighted photo marker.
- **False Call State:** When a photo is marked as a False Call, its marker colour on the map should update immediately to reflect the human override (e.g., a distinct outline or icon).
- **Z-Index Management:** The side panel must sit above the map canvas. The static overlay panels (Left, Bottom, Right from Level 1) should be hidden or collapsed when the side panel is open to avoid visual clutter.
